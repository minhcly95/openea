ea::EvolutionStrategy Class Reference

Detailed Description

A trivial implementation of Evolution Strategy.

This class implements a trivial \((\mu + \lambda)\)-ES or \((\mu,\lambda)\)-ES with basic operators including recombination, mutation, parent and survival selection. The execution followed this order:

  • [S] Spawning phase: Recombinator in recombination list is applied on the main pool to spawn new Genome and put into the spawn pool.
  • [M] Mutation phase: Mutator in mutation list is applied on newly created Genome.
  • [E] Evaluation phase: Evaluator is applied to each Genome, creates a Organism wrapper for each Genome.
  • [F] Filtering phase: Survival Selector is applied to truncate the main pool to the population size.
  • Global Hook execution.

The numbering of Pool of this Strategy is:

  • Main pool: OrganismPool #0 containing evaluated and filtered Organism.
  • Spawn pool: GenomePool #1 containing newly spawned and mutated Genome.

To add operators into the DefaultStrategy, use these class fields:

  • initializer
  • evaluator
  • recombinators
  • mutators
  • survivalSelector
  • hooks

To switch between \((\mu + \lambda)\) (which merges new Organism with the old main pool) and \((\mu,\lambda)\) (which discard the old main pool and keep only new Organism), see SetSelectionMode().

void ea::EvolutionStrategy::SetSelectionMode ( SelectionMode  pMode)

Set the selection mode (plus or comma) of the algorithm.

This function allows users to switch between the \((\mu+\lambda)\)-ES and \((\mu,\lambda)\)-ES algorithm:

  • If \((\mu+\lambda)\)-ES is used (PLUS), the old pool will be kept and merge with new Organism.
  • If \((\mu,\lambda)\)-ES is used (COMMA), the old pool will be discarded, which requires the total offspring ratio must be greater than 1.
    Parameters
    pModeThe selection mode (PLUS or COMMA).
